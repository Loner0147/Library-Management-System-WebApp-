A simple library web app for universities... That support complete automation of library management operations

ROLES: USER(STUDENT) AND ADMIN (LIBRARY STAFF)
FEATURES (Based on Roles):

USER(STUDENT)
 Borrow/return books;
 View book availability (Number of copies available)
 Have reading access to books available in the library system's database. (i.e for digitally available books) 

ADMIN (LIBRARY STAFF)
 Manage book and user records. 
 Approve/reject borrow requests from users (students);
 Create, delete, modify books and book information;
 Promote a user to admin role/demote an admin to user role;
 Modify user information; 
 Create/ delete user;

TECHNOLOGIES USED:
 Backend: JAVA (SPRINGBOOT)
 Forntend: HTML, CSS & JAVASCRIPT
 End-to-End Integration: THYMELEAF (SpringBoot)
 Other tools; DOCKER, WINDSURF IDE, REDINARY CLOUD...
